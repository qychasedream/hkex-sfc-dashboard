# 香港监管资讯看板 — 自动更新修复记录

## 日期
2026年7月6日

## 问题
用户反馈：GitHub Actions 设定的每天早上8点自动更新没有在 GitHub Pages 网站上生效。

## 诊断

### 根本原因
GitHub Pages 部署源配置为 `/docs` 目录，只服务 `docs/` 下的文件。但自动化工作流 `daily-update.yml` 把数据写入**根目录的 `data/`**，该目录不在 Pages 服务范围内。

```
仓库结构:
├── data/              ← 工作流更新这里 ✗ Pages 不服务
│   ├── update_meta.json
│   ├── sfc_news.json
│   └── ...
├── docs/              ← Pages 只服务这里 ✓
│   ├── index.html
│   └── regulatory/
│       └── index.html  → JS 加载 ../../data/ → 404!
```

### 其他问题
| 问题 | 说明 |
|------|------|
| `docs/index.html` 数据写死 | 第296行静态嵌入 REGULATORY_DATA，永远不更新 |
| `docs/regulatory/index.html` 路径错误 | `../../data/` 指向 Pages 不服务的路径 |
| Header 硬编码日期 | "数据更新于 2026年7月5日" 与底部动态时间不一致 |

## 2026-07-06 第二次修复：HKEX数据源大幅扩展

### 问题
用户反馈：
1. 联交所大量资讯没有被抓取（如"刊發及更新有關結構性產品發行人的指引"）
2. HKEX 指引更新只有6条，远少于实际

### 根因
`update_data.py` 的 HKEX 数据采集范围严重不足：
- **仅从 cn-rules.hkex.com.hk 的18个主题页面抓取FAQ条目**
- **完全缺失**：HKEX 监管通讯 (Regulatory Announcements)、新闻稿 (News Releases)
- **指引信 (Guidance Letters)**：虽然存在于 cn-rules.hkex.com.hk 但被过滤条件跳过
- 过滤条件 `re.match(r'^\d+\.\d+\s', ref)` 误杀了很多条目

### 修复内容

#### 1. 新增 `fetch_hkex_regulatory_announcements()` — HKEX 监管通讯 RSS
- RSS 源：`https://www.hkex.com.hk/Services/RSS-Feeds/regulatory-announcements?sc_lang=zh-HK`
- 自动分类：纪律行动→disciplinary, 咨询文件→consultation, 除牌→enforcement_news, 规则修订→regulatory_rule
- 输出：`hkex_regulatory_announcements.json`（49条）
- **捕获了"聯交所就優化結構性產品上市制度建議刊發諮詢總結"等关键公告**

#### 2. 新增 `fetch_hkex_news_releases()` — HKEX 新闻稿 RSS
- RSS 源过滤监管相关关键词（上市/监管/合规/指数/期货等）
- 输出：`hkex_news_releases.json`（43条监管相关新闻稿）

#### 3. 修复 `fetch_hkex_guidance()` — 同时捕获 FAQ + 指引信
- 移除过度的 ref 过滤条件
- 新增 GL-xxx 格式识别（指引信）
- 新增 `docType` 字段区分 `faq` 和 `guidance_letter`
- 日期范围从当年扩展到当年+去年
- 结果：从 **6条 → 49条**（31条指引信 + 18条FAQ）

#### 4. 前端适配
- `docs/index.html`：加载并展示新的2个数据源
- 新增 `transformHkexRegulatory()` 和 `transformHkexNews()` 转换函数
- Footer 统计增加 HKEX 监管通讯数量
- 回退显示上限从 50 → 100
